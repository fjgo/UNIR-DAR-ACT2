Despliegue

Una vez que tu aplicación está lista, puedes desplegarla en un servidor o servicio en la nube. Angular CLI facilita el proceso de compilación y optimización del código.

Compilar la aplicación

Ejecuta el siguiente comando para generar los archivos de producción en la carpeta dist: 

ng build --prod

Esto optimiza el código, comprimiendo los archivos y eliminando partes innecesarias.

Esto crea una carpeta dist/ con los archivos necesarios para producción. Luego, puedes usar servicios como Firebase Hosting o GitHub Pages para hospedar los archivos generados.

Desplegar en un servidor estático

Puedes copiar los archivos generados en dist a cualquier servidor web estático como Apache, Nginx o servicios como GitHub Pages.

Ejemplo con Nginx

Configura el servidor para servir el archivo index.html:

server {
  listen 80;
  server_name ejemplo.com;

  location / {
    root /ruta/a/dist/;
    index index.html;
    try_files $uri $uri/ /index.html;
  }
}

Despliegue en Firebase Hosting

Firebase Hosting es una opción rápida y gratuita para desplegar aplicaciones Angular:

  1. Instala Firebase CLI:

    npm install -g firebase-tools

  2. Autentica Firebase e inicializa el proyecto:

    firebase login
firebase init

    Selecciona Firebase Hosting y proporciona la ruta de la carpeta dist.

  3. Despliega la aplicación:

    firebase deploy

    4. Despliegue en GitHub Pages

    GitHub Pages es un servicio gratuito que permite desplegar sitios web estáticos directamente desde un repositorio de GitHub.

    1. Crea un nuevo repositorio en GitHub:

    2. Ve a https://github.com/new y crea un nuevo repositorio.

    3. Asigna un nombre y una descripción para el repositorio.

    4. Crea una nueva rama llamada gh-pages en el repositorio.

    5. Configura el repositorio para desplegar la aplicación en GitHub Pages seleccionando la rama gh-pages como fuente.

    6. Añade el módulo angular-cli-ghpages de Angular para desplegar la aplicación en GitHub Pages: 
      ng add angular-cli-ghpages

    7. Configura tu repositorio local para conectarlo con el repositorio de GitHub. 

      git remote add origin https://github.com/nombre-usuario/nombre-repositorio.git
git branch -M main
git push -u origin main
    8. Despliega la aplicación en GitHub Pages: 
      ng deploy --base-href=/nombre-repositorio/
