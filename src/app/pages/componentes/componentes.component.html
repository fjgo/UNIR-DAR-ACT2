Componentes

Angular se basa en una arquitectura de componentes. Cada componente es una pieza independiente de la interfaz de usuario y contiene su propia lógica y vista. A continuación, aprenderemos a crear componentes en Angular:

Generar un componente: Utiliza Angular CLI para crear un nuevo componente. El siguiente comando generará automáticamente los archivos necesarios: 

ng generate component nombre-componente

Esto creará una carpeta con los archivos nombre-componente.component.ts, nombre-componente.component.html, nombre-componente.component.css y nombre-componente.component.spec.ts.

Estructura de un componente:

  • TypeScript (.ts): Incluye la lógica y los datos del componente. Este archivo define una clase que contiene las propiedades y métodos utilizados en la plantilla HTML.
  • HTML (.html): Define la plantilla que se renderizará.
  • CSS/SCSS (.css): Contiene los estilos aplicados al componente. Tambien puedes usar SCSS o LESS segun la configuracion del proyecto.
  • Pruebas (.spec.ts): Permite realizar pruebas unitarias. Angular CLI genera automáticamente un archivo .spec.ts para cada componente.

Uso del componente: Una vez generado, puedes usar el selector del componente en cualquier archivo HTML. Por ejemplo: 

<app-nombre-componente></app-nombre-componente>

Comunicación entre componentes: Angular permite la comunicación mediante @Input y @Output para pasar datos y eventos entre componentes padre e hijo.

Además, los componentes pueden comunicarse entre sí utilizando Input y Output. Esto permite pasar datos desde un componente padre a uno hijo y viceversa. Por ejemplo:

En el componente padre:

<app-inicio [titulo]="'Bienvenido'"></app-inicio>

En el componente hijo:

@Input() titulo: string;

Esta interacción entre componentes es esencial para construir aplicaciones dinámicas y modulares.
